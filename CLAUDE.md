# colorful-servers

竹内貴紀が管理する VPS サーバー群の設定記録リポジトリ。

## 目的

- 各サーバーの設定・運用状況を Markdown で記録する
- 新しいセットアップや変更作業を Claude Code で補助する
- 過去の作業履歴を参照しやすくし、メンテナンスを効率化する

## ディレクトリ構造

トップレベルにホスト名（サブドメイン部分）のディレクトリを置く。

```
colorful-servers/
├── CLAUDE.md
├── README.md
├── .gitignore
└── {hostname}/       # 例: sumi, kurenai, hanada ...
    └── README.md     # そのサーバーの概要・現状
```

## ネーミング規則

- ドメイン: `*.colorful-servers.com`
- ホスト名: 和色（日本の伝統的な色名）を使う
  - 例: sumi（墨）, kurenai（紅）, hanada（縹）, kou（紅）

## シークレット管理方針

このリポジトリはプライベートだが、SSH 鍵・パスワード・API トークン・IPアドレスなどの秘密情報は**平文でコミットしない**。

- 秘密情報は sops / age / git-crypt などの暗号化ツールで暗号化してからコミットする
- または 1Password などのパスワードマネージャーで管理し、リポジトリには保存しない
- `.gitignore` に記載されたパターンのファイルは絶対にコミットしない

## Claude Code への作業指示

### 記録作業時

- サーバーの設定変更や新規セットアップを記録する際は、対象ホストのディレクトリに Markdown ファイルを追加・更新する
- 変更内容と理由を明記する（「何をしたか」だけでなく「なぜそうしたか」も記録）
- コマンドや設定ファイルの内容はコードブロックで記載する

### リサーチ作業時

- パッケージのバージョンや推奨設定など、時間とともに変化する情報は最新情報を Web 検索して確認する
- セキュリティ上の推奨事項（TLS 設定、SSH 設定など）は現時点のベストプラクティスを調べてから提案する

### 禁止事項

- 秘密情報（IPアドレス、パスワード、APIキー、SSH 秘密鍵など）を平文でファイルに書かない
- 暗号化されていない秘密情報をコミットしない
