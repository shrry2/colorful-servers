# wrap — ドキュメント同期スキル

colorful-servers リポジトリでの作業区切りに呼び出す。
会話を振り返り、変更内容をドキュメントとメモリに反映してから commit + push する。

## ステップ 1：作業内容を把握する

会話履歴を振り返り、今回の作業で何が変わったかを整理する。

- 新しくセットアップしたサービス・ソフトウェア
- 変更・更新した設定（compose.yaml、設定ファイルなど）
- 解決した問題とその原因・解決策
- 削除したもの（サービス、ファイル、設定）
- 計画段階から完了に変わったもの

## ステップ 2：現在のドキュメントを読む

変更の影響を受けるファイルを Read ツールで確認する。

確認対象：
- `{hostname}/README.md` — サーバー概要・稼働サービス一覧・変更履歴
- `{hostname}/{service}.md` — 変更があったサービスのドキュメント

リポジトリの構造：
```
colorful-servers/
├── CLAUDE.md
├── README.md
└── {hostname}/          # 和色の名前（sumi, kurenai, hanada ...）
    ├── README.md        # サーバー概要・稼働サービス・変更履歴
    └── {service}.md     # サービス固有の設定記録（postgres.md, pgbouncer.md など）
```

## ステップ 3：メモリを確認する

`/Users/takaki/.claude/projects/-Users-takaki-ghq-github-com-shrry2-colorful-servers/memory/MEMORY.md` を読み、
更新が必要なメモリを特定する。

## ステップ 4：ドキュメントを更新する

現状と乖離している箇所を更新する。

**`{hostname}/README.md` の更新ポイント：**
- 稼働中のサービス一覧（追加・削除）
- インストール済みソフトウェア
- 計画中の作業（完了したものを消す）
- 変更履歴に今日の作業を追記（日付は `date +%Y-%m-%d` で確認）

**`{hostname}/{service}.md` の更新ポイント：**
- ディレクトリ構成（ファイルの追加・変更）
- 設定ファイルの内容（compose.yaml、ini ファイルなど）
- 手順・コマンド
- 「計画中」→ 実際の設定に書き直す

**新規ファイルの作成：**
新しいサービスを追加した場合は `{hostname}/{service}.md` を作成する。

## ステップ 5：メモリを更新する

以下の場合にメモリを更新または追加する：

- 将来の作業に影響する設計判断（なぜそうしたか）
- 再現しうるハマりポイントと解決策
- サーバー固有の重要な特性や制約

コードや設定で表現できること（コマンド、設定値、ファイルパス）はドキュメントに書く。
メモリには「なぜ」「どんな判断をしたか」を書く。

## ステップ 6：秘密情報の確認

コミット前に以下が平文でファイルに含まれていないか確認する：
- IPアドレス（メモリには OK、コミット対象ファイルは NG）
- パスワード・APIトークン・秘密鍵

`.gitignore` に記載のパターンに該当するファイルがステージングされていないかも確認。

## ステップ 7：commit + push

```bash
git -C /path/to/repo add ファイル名...   # 変更したファイルを個別に指定
git -C /path/to/repo status              # 確認
git -C /path/to/repo commit -m "..."
git -C /path/to/repo push
```

コミットメッセージは変更内容を日本語で簡潔に記述する。
複数の変更がある場合は主要な変更を1行目に、詳細を本文に書く。

Co-Authored-By 行を忘れずに含める：
```
Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

## ドキュメントを書く際の基準

**書くもの：**
- 現在の状態（どうなっているか）
- 重要なコマンドと設定
- ハマりやすいポイントと注意事項（将来また困りそうなもの）
- 「計画中」「予定」は現実に合わせて更新する

**書かないもの：**
- 過去の経緯（なぜ変えたか）— 変更履歴に日付と概要だけ
- 試行錯誤の過程
- 秘密情報（IPアドレス、パスワード、秘密鍵）
